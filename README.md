# AtCoder ステップアップ支援

灰 → 茶 → 緑 → 水 と上がるために、**問題選び**と**発想（解法パターンの引き出し）**の両方を補助するローカル Web アプリです。

## 機能

- **はじめに** — AtCoder が完全に手つかずの人向け。アカウント作成→言語選び→最初の問題→ABC参加…までの最初の手順（チェックリスト・自動保存）、言語の選び方、よくある疑問。初期表示タブ。
- **診断** — AtCoder のユーザー名から、AC した問題の難易度分布・現在の色・次の目標を表示。さらに **到達レート目安** — 「この難易度を安定して解ける」から内部レートを数値で推定（AtCoder と同じ Elo 型ロジスティック：到達レート ≈ 難易度 + 400·log₁₀(p/(1−p))）。
- **問題リコメンド** — 今の実力の少し上の難易度帯から、まだ AC していない問題を厳選（ABC 優先）。
- **発想補助**
  - どんな問題にも効く「発想の3ステップ」（制約→分類→パターン照合）
  - 解法カテゴリ別の段階ヒント（全探索 / bit全探索 / 累積和 / 二分探索 / 貪欲 / DP / BFS・DFS / Union-Find / 数学 / しゃくとり）。各カテゴリに練習問題・問題集リンク（EDPC / 競技プログラミングの鉄則 / 典型90問 等）付き
  - **AI ヒント** — 問題文を貼ると、ネタバレを避けた段階ヒントを生成（Claude API、任意）
- **ロードマップ** — 色ごとに身につけたい手筋のチェックリスト（自動保存）。完了バーと、その目標難易度の問題へジャンプするボタン付き。

データは [AtCoder Problems (kenkoooo)](https://kenkoooo.com/atcoder/) の公開 API を使用します。

## 起動方法

ES Modules を使うため、`file://` で直接開くのではなく**ローカルサーバ経由**で開きます。

### かんたん（Windows）
`start.bat` をダブルクリック。ブラウザで `http://localhost:8000/` が開きます。
（Python が必要です。なければ下の方法を使ってください）

### Python がある場合
```
cd atcoder-stepup
python -m http.server 8000
```
→ ブラウザで http://localhost:8000/

### VS Code の場合
拡張機能 **Live Server** を入れて、`index.html` を右クリック →「Open with Live Server」。

## AI ヒントの設定（任意）

1. 「設定」タブで **Anthropic API キー**（`sk-ant-...`）を入力して保存。
2. モデルを選択（既定は最高精度の **Opus 4.8**。コスト重視なら Haiku 4.5）。
3. 「発想」タブの AI ヒント欄に問題文を貼って生成。

- API キーは**サーバーには送られず**、このブラウザ（localStorage）に保存され、Anthropic API へ直接送信されます。
- 共有 PC では使用後に「キーを消去」してください。
- 利用は従量課金です（Anthropic のアカウントが必要）。

## 仕組み・構成

```
atcoder-stepup/
├─ index.html
├─ start.bat
├─ css/styles.css
└─ js/
   ├─ config.js       色帯・API URL・モデル定義
   ├─ colors.js       難易度→色帯の変換、表示難易度補正
   ├─ storage.js      localStorage ラッパ
   ├─ api.js          AtCoder Problems からの取得（問題/難易度/提出）
   ├─ diagnose.js     AC 分布・現在地の推定
   ├─ recommend.js    難易度帯からの未AC厳選
   ├─ ai.js           Claude API による段階ヒント生成
   ├─ app.js          UI 配線
   └─ data/
      ├─ roadmap.js    色別ロードマップ
      └─ categories.js 発想の手順・カテゴリ別ヒント
```

## メモ

- 難易度の色境界：灰 <400 / 茶 400 / 緑 800 / 水 1200 / 青 1600 …
- 現在の色は、レーティングを入力すればそれを優先。未入力なら「解いた問題の上位1割の難易度帯」で推定します。
- 初回の問題データ取得は数 MB あります（メモリにキャッシュ。「設定 > 問題データを再取得」で更新）。
